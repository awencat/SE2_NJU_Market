# 交付项 3-4 测试说明

## 交付项 3：单元测试代码

本次补充的单元测试位于：

- `backend/src/test/java/com/marketback/main/service/OrderEntityServiceImplTest.java`
- `backend/src/test/java/com/marketback/main/service/RatingServiceImplTest.java`

覆盖的核心业务规则：

| 测试类 | 覆盖模块 | 重点场景 |
| --- | --- | --- |
| `OrderEntityServiceImplTest` | 订单购买与取消 | 正常购买、扣减库存、生成订单明细、购买数量边界、不能购买自己的商品、超库存购买、取消订单恢复库存、非买家取消拦截 |
| `RatingServiceImplTest` | 商品评分 | 已购买商品可评分、评分范围 1-5、未购买不可评分、重复评分拦截、评分平均值统计 |

本次同时补齐了服务层边界校验，避免业务规则只依赖 Controller 参数注解：

- `OrderEntityServiceImpl.purchase`：校验 `buyerId`、`goodId`、`count`，并拒绝 `count < 1`。
- `RatingServiceImpl.save`：校验评分实体非空，并拒绝 `score` 不在 1-5 范围内。

JaCoCo 覆盖率记录：

| 核心类 | 行覆盖率 | 分支覆盖率 | 说明 |
| --- | ---: | ---: | --- |
| `OrderEntityServiceImpl` | 68 / 98 = 69.4% | 26 / 48 = 54.2% | 覆盖购买、取消、库存和权限边界 |
| `RatingServiceImpl` | 36 / 38 = 94.7% | 16 / 22 = 72.7% | 覆盖评分提交、重复评分、购买校验和汇总 |

> 全项目仍包含大量实体类、CRUD 控制器和图片上传等历史未测模块，因此本说明按 Phase 4 要求中的“核心模块”口径记录覆盖率。

## 交付项 4：集成测试代码

集成测试位于：

- `backend/src/test/java/com/marketback/main/controller/CoreBusinessFlowIntegrationTest.java`

覆盖流程：

| 类型 | 用例 | 验证点 |
| --- | --- | --- |
| 完整正常流程 | 用户注册 -> 登录 -> 发布商品 -> 购买 -> 评价 | 接口可串联调用，返回结构符合 `ApiResponse`，订单状态为 `PENDING`，交易金额计算正确，评分保存成功 |
| 异常流程 1 | 购买数量为 0 | Controller 参数校验返回 `count must be at least 1` |
| 异常流程 2 | 重复评价同一商品 | 业务校验返回 `user already rated this good` |
| 异常流程 3 | 登录密码错误 | 认证接口返回 `username or password is incorrect` |
