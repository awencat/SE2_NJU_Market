# 交付项 9：演示说明

## 1. 演示目标

本演示用于展示校园二手交易平台的核心可运行版本。重点覆盖普通用户注册登录、商品浏览、商品详情、订购退订、评论评分、举报，以及管理员相关接口能力。

本项目当前采用前后端分离结构：

- 后端：Spring Boot 3、MyBatis-Plus、MySQL，默认端口 `8080`
- 前端：Vue 3、Vite、Element Plus，默认端口 `5173`
- 前端通过 Vite 代理把 `/api` 请求转发到 `http://localhost:8080`

## 2. 运行环境

| JDK | 17 |

## 3. 数据库准备

先在 MySQL 中创建数据库：

```sql
CREATE DATABASE IF NOT EXISTS nju_market DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

然后按顺序执行：

```text
backend/src/main/resources/sql/schema.sql
backend/src/main/resources/sql/data.sql
```

`schema.sql` 用于建表，`data.sql` 用于插入演示数据。

如果使用 Navicat，可以按以下方式操作：

1. 连接本机 MySQL。
2. 新建数据库 `nju_market`。
3. 打开 SQL 窗口。
4. 先复制并执行 `schema.sql`。
5. 再复制并执行 `data.sql`。
6. 执行后检查 `user`、`admin`、`good`、`order`、`comment`、`rating` 等表是否有数据。

## 4. 数据库连接配置

后端配置文件位于：

```text
backend/src/main/resources/application.yaml
```

当前配置优先读取环境变量：

```yaml
spring:
  datasource:
    url: ${DB_URL:jdbc:mysql://localhost:3306/nju_market?useUnicode=true&characterEncoding=UTF-8&serverTimezone=Asia/Shanghai}
    username: ${DB_USERNAME:root}
    password: ${DB_PASSWORD:root}
```

如果本机 MySQL 密码不是 `root`，可以在 PowerShell 中临时设置：

```powershell
$env:DB_URL='jdbc:mysql://localhost:3306/nju_market?useUnicode=true&characterEncoding=UTF-8&serverTimezone=Asia/Shanghai'
$env:DB_USERNAME='root'
$env:DB_PASSWORD='你的数据库密码'
```

如果希望长期保存，可以使用：

```powershell
setx DB_USERNAME "root"
setx DB_PASSWORD "你的数据库密码"
```

注意：`setx` 对已经打开的终端不立即生效，需要重新打开 PowerShell 或 IDE 终端。

## 5. 启动后端

进入后端目录：

```powershell
cd backend
.\mvnw.cmd spring-boot:run
```

看到类似日志表示后端启动成功：

```text
Tomcat started on port 8080 (http) with context path '/'
Started DemoApplication
```

可以用浏览器访问：

```text
http://localhost:8080
```

如果返回 `No static resource`，不代表后端没启动。后端主要提供 API，不提供首页静态页面。

## 6. 启动前端

新开一个终端，进入前端目录：

```powershell
cd frontend
npm install
npm run dev
```

启动后访问：

```text
http://localhost:5173
```

前端请求 `/api` 时会自动代理到后端 `8080`。

## 8. 核心演示路径

### 8.1 用户登录

1. 打开 `http://localhost:5173`。
2. 使用 `alice / 123456` 或 `bob / 123456` 登录。
3. 登录成功后进入首页。

验证点：

- 登录接口返回成功。
- 前端能保存当前用户信息。
- 页面能正常进入首页或个人主页。

### 8.2 商品浏览和详情页

1. 登录后进入日用商城。
2. 查看商品卡片列表。
3. 点击任意商品卡片进入商品详情页。

验证点：

- 商品列表能正常加载。
- 商品卡片显示名称、价格、卖家和状态。
- 商品详情页显示商品描述、卖家联系方式、评分和评论。

### 8.3 订购商品

1. 在详情页选择一个状态为 `ON_SALE` 的商品。
2. 点击订购商品。
3. 订购成功后查看弹窗中的订单号、金额、卖家电话或邮箱。
4. 回到商品列表，确认该商品状态变为 `RESERVED`。

验证点：

- 后端创建订单和订单项。
- 商品状态从 `ON_SALE` 变为 `RESERVED`。
- 买家能看到卖家联系方式。

注意：买家不能购买自己发布的商品。如果要演示订购，建议切换到非卖家账号操作。

### 8.4 个人主页查看订单和卖家通知

1. 登录后进入个人主页。
2. 查看买家订单区域。
3. 如果当前用户是卖家，查看收到的订单通知。
4. 对买家订单执行退订。

验证点：

- 买家能看到自己的订单。
- 卖家能看到商品被订购的信息。
- 退订后订单状态和商品状态按后端逻辑更新。

### 8.5 评论和评分

1. 进入已购买过的商品详情页。
2. 在评论区域输入评论并提交。
3. 在评分区域选择星级并提交。
4. 刷新页面，确认评论和评分显示正常。

验证点：

- 已购买用户可以评论和评分。
- 未购买用户提交评论或评分会被后端拒绝。
- 同一个用户不能对同一个商品重复评分。
- 商品详情页显示平均分和评分人数。

### 8.6 举报商品

1. 进入商品详情页。
2. 切换到举报区域。
3. 输入举报原因并提交。
4. 在数据库 `report` 表中查看新增记录。

验证点：

- 举报请求能成功提交。
- 举报状态默认为 `PENDING`。
- 举报记录可供管理员系统后续处理。

### 8.7 管理员能力演示

当前管理员接口主要在后端提供，接口包括：

```text
POST /api/admins/login
GET /api/users
POST /api/users/{id}/ban
POST /api/users/{id}/unban
GET /api/goods
GET /api/reports
```

可用 Postman、Apifox 或浏览器开发者工具演示：

1. 使用 `root / 123456` 调用管理员登录接口。
2. 查询用户列表。
3. 对某个用户执行封禁。
4. 再执行解封。
5. 查询商品列表和举报列表。

验证点：

- 管理员账号能登录。
- 管理员能查看用户和商品。
- 用户封禁后不能正常登录。
- 解封后用户可恢复登录。

## 9. 异常流程演示

建议演示以下异常场景：

| 场景 | 操作 | 预期结果 |
| --- | --- | --- |
| 密码错误 | 使用正确用户名和错误密码登录 | 返回登录失败提示 |
| 购买自己的商品 | 卖家购买自己发布的商品 | 后端拒绝，提示不能购买自己的商品 |
| 购买不可售商品 | 购买 `RESERVED` 或 `SOLD` 商品 | 后端拒绝，前端提示失败 |
| 未购买评论 | 未购买用户直接评论商品 | 后端拒绝，前端提示只有购买后可评论 |
| 重复评分 | 同一用户对同一商品评分两次 | 后端拒绝重复评分 |

## 12. 常见问题

### 12.1 前端显示 server error

优先检查后端控制台日志。如果后端没有明显报错，再检查浏览器 Network 中真正失败的接口。

常见原因：

- 数据库没启动。
- 数据库密码配置不正确。
- `schema.sql` 或 `data.sql` 没有执行。
- 当前商品状态不允许购买。
- 当前用户没有购买过该商品，却提交了评论或评分。

### 12.2 后端显示 Tomcat started，但页面仍然打不开

`http://localhost:8080` 不是前端页面地址。前端页面地址是：

```text
http://localhost:5173
```

后端 `8080` 只负责 API。

### 12.3 设置了 setx 但密码仍然不生效

`setx` 只影响新打开的终端。需要关闭当前终端，重新打开后再启动后端。

也可以在当前 PowerShell 里临时设置：

```powershell
$env:DB_PASSWORD='你的数据库密码'
```
