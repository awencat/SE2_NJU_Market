用例 1：发布商品
---------

| 项目       | 内容                                                                                                                                                                                                                                                                                         |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **用例名称** | 发布商品                                                                                                                                                                                                                                                                                       |
| **参与者**  | 已认证的卖家（普通学生）                                                                                                                                                                                                                                                                               |
| **前置条件** | 1. 用户已登录并完成实名认证<br>2. 用户信用分 ≥ 60<br>3. 用户当前上架商品数量 < 20（平台限制）                                                                                                                                                                                                                               |
| **基本流**  | 1. 卖家点击“发布商品”按钮<br>2. 系统显示发布表单，包含：标题、类别、描述、原价、售价、新旧程度（全新/几乎全新/有使用痕迹）、图片上传（至少1张）<br>3. 卖家填写所有必填项并上传图片<br>4. 卖家点击“提交”<br>5. 系统校验数据（价格范围、违禁词、图片格式）<br>6. 系统生成商品ID，状态设为“待售”<br>7. 系统将商品展示在商品列表首页<br>8. 系统提示“发布成功”                                                                              |
| **替代流**  | **3a. 卖家中途取消**：点击返回或关闭 → 用例结束，不保存<br>**5a. 售价超出0.01-5000元范围**：系统提示“售价必须在0.01~5000元之间” → 返回步骤2<br>**5b. 描述包含违禁词（如烟酒）**：系统提示“描述含有违规内容，请修改” → 返回步骤2<br>**5c. 图片少于1张**：系统提示“请至少上传1张图片” → 返回步骤2<br>**5d. 信用分不足60**：系统提示“信用分过低，无法发布商品” → 用例结束<br>**5e. 已上架数量达到20**：系统提示“已达到上架上限，请先下架部分商品” → 用例结束 |
| **后置条件** | 1. 商品信息存储到数据库<br>2. 卖家“我的发布”列表中增加该商品<br>3. 商品出现在首页及搜索结果中                                                                                                                                                                                                                                   |

用例 2：购买/联系卖家（发起交易意向）
--------------------

> 注：该用例的核心是买家通过私信联系卖家并表达购买意向，不涉及在线支付。

| 项目       | 内容                                                                                                                                                                                                                                                         |
| -------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **用例名称** | 联系卖家                                                                                                                                                                                                                                                       |
| **参与者**  | 已认证的买家（普通学生）                                                                                                                                                                                                                                               |
| **前置条件** | 1. 买家已登录并完成实名认证<br>2. 存在一个“待售”状态的商品<br>3. 买家不是该商品的卖家本人                                                                                                                                                                                                     |
| **基本流**  | 1. 买家浏览商品列表或搜索，找到目标商品<br>2. 买家点击商品进入详情页<br>3. 买家点击“联系卖家”按钮<br>4. 系统打开私信聊天界面，自动填充商品标题作为引用<br>5. 买家输入文字（如“请问还在吗？想面交”）<br>6. 买家发送消息<br>7. 系统保存消息，并推送站内信通知卖家<br>8. 系统显示“发送成功”                                                                                  |
| **替代流**  | **3a. 商品状态已变为“已预订”或“已售”**：系统提示“该商品暂时无法联系，可能已被预订或售出” → 返回商品详情页<br>**5a. 买家未输入任何文字**：系统提示“请输入消息内容” → 返回步骤5<br>**5b. 买家输入敏感词**：系统过滤并提示“含有不当用语，请修改” → 返回步骤5<br>**6a. 卖家已封禁或信用分过低**：系统提示“对方账号异常，无法发送消息” → 用例结束<br>**6b. 买家当天发送消息超过30条**：系统提示“操作频繁，请稍后再试” → 用例结束 |
| **后置条件** | 1. 私信记录保存到数据库<br>2. 卖家收到未读消息提醒<br>3. 聊天界面显示最新消息                                                                                                                                                                                                            |

用例 3：确认收货并评价
------------

> 注：本用例假设买卖双方线下完成交易后，买家在平台上点击“确认收货”，然后双方互评。

| 项目       | 内容                                                                                                                                                                                                                                                 |
| -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **用例名称** | 确认收货并评价                                                                                                                                                                                                                                            |
| **参与者**  | 买家（也可扩展卖家，但核心是买家确认）                                                                                                                                                                                                                                |
| **前置条件** | 1. 买家已登录<br>2. 存在一笔“已预订”或“待确认”的交易（双方已通过私信约定成交）<br>3. 卖家已将商品状态手动改为“已售”或买家可触发确认                                                                                                                                                                      |
| **基本流**  | 1. 买家进入“我的订单”或“我买的”列表<br>2. 找到对应商品，点击“确认收货”按钮<br>3. 系统弹出二次确认：“确认已收到商品且无纠纷？”<br>4. 买家点击“确认”<br>5. 系统将交易状态更新为“已完成”<br>6. 系统自动发送站内信通知卖家“买家已确认收货”<br>7. 系统提示“交易完成，去评价”<br>8. 买家点击“评价”，选择1-5星，填写文字评价（可选）<br>9. 系统保存评价，更新卖家信用分<br>10. 卖家收到评价通知，也可对买家进行评价 |
| **替代流**  | **2a. 商品状态不是“待确认”**：系统提示“当前状态无法确认收货” → 返回列表<br>**4a. 买家点击“取消”**：关闭弹窗，用例结束<br>**8a. 买家跳过评价**：点击“暂不评价”，用例直接结束，后续可补评<br>**8b. 买家评价后想修改**：系统不允许修改（评价一旦提交不可修改）<br>**纠纷场景（扩展）**：若买家实际未收到货或商品不符，应在步骤3之前发起“投诉”，不执行确认收货         |
| **后置条件** | 1. 交易状态变为“已完成”<br>2. 买家的信用分微增（如+1），卖家信用分根据评价增减<br>3. 评价记录展示在双方个人主页<br>4. 该商品从“待售”列表中彻底移除                                                                                                                                                           |
