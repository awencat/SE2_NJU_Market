| 对比维度            | AI 生成的需求                                                | 人工调研发现的需求                                           | 分析                                                         |
|---------|-------------|----------------|------|
| AI 正确覆盖的需求   | 1. 校园身份认证 2. 商品发布与管理 3. 商品分类、搜索、排序 4. 端内实时聊天 5. 交易评价与信用机制 | 问卷明确支持这些核心方向：**校内实名/学生认证**支持率 **86.36%**，且 **90.91%** 的用户愿意使用校内统一身份认证提升安全；**商品分类/搜索/排序**支持率 **72.73%**；**内置即时聊天**支持率 **68.18%**；**交易评价与信用分**支持率 **59.09%**，且 **100%** 的受访者表示信用系统会影响交易决策；商品发布时用户最重视的信息包括**商品图片（95.45%）**、**原价/期望价格（90.91%）**、**联系方式（77.27%）**、**所在校区/宿舍区（72.73%）**等。 | 这部分说明 AI 抓住了校园二手平台最核心的主线：**安全、发布、检索、沟通、信用**。整体方向是对的，可以保留作为 SRS 主干。问题在于 AI 有时把“用户需求”直接写成了“实现方案”，导致粒度偏技术化。 |
| AI 遗漏的需求       | 1. 未突出“按校区/宿舍楼筛选” 2. 未写“求购板块” 3. 未写“赠送/交换专区” 4. 未充分体现“举报/纠纷处理/管理员介入” 5. 未体现“校内快递/小助手配送、跨校区中转/代送” 6. 未强调“联系方式可匿名” 7. 未突出校园高频类目优先级 | 问卷里这些都是明显需求：**按校区/宿舍楼筛选商品**支持率 **63.64%**；**求购发布**在功能题中有 **50%** 支持，而单独问是否需要求购板块时，**90.91%** 选择“非常需要/比较需要”；**赠送/交换专区**支持率 **59.09%**；**管理员审核/举报机制**支持率 **45.45%**，且 **72.73%** 的用户认为举报与纠纷处理必须存在并且要有管理员介入；交易方式上，用户希望支持**同校区面交（59.09%）**、**跨校区代送/中转（54.55%）**、**校内快递/小助手配送（63.64%）**；发布信息中明确提到**联系方式可匿名**；商品类目中 **教材/教辅/笔记（100%）**、**生活用品（81.82%）**、**电子产品/体育用品（72.73%）**最受关注。 | AI 更像是在套一个“通用二手交易平台模板”，所以容易抓到通用需求，却漏掉**校园场景特有需求**。尤其是**校区地理维度、求购、赠送交换、纠纷治理、校内配送**这些，正是人工调研最有价值的部分，也是平台与闲鱼、转转区别最大的地方。AI 会遗漏，主要因为这些需求依赖具体场景，而不是通用电商常识。 |
| AI 捏造的需求       | 1. 线上支付 / 微信支付宝支付 SDK 2. 订单系统中的“待付款/待面交、交易中、确认收货、库存锁定” 3. 留言板、商品收藏、推荐算法 4. 明确支持“小程序/App + Web 端” 5. “最多上传 9 张图片” 6. “手机号注册 + edu邮箱/学生证人工审核” 7. 1.5 秒响应、1000 并发、500KB 图片、99.9% 可用性 8. 指定 Spring Boot / Go / Flutter / Redis / MySQL 等技术选型 | 问卷并没有直接问到：是否需要**线上支付**、是否需要**完整订单流转**、是否需要**收藏/留言板**、是否一定要做 **Web 端**、最多传几张图、是否必须手机号注册、系统性能指标、技术架构选型等。人工调研中真正被明确提到的是：**身份可信、搜索高效、沟通方便、信用可参考、交易方式适配校园、求购/赠送/举报要有**。 | 这些内容大多不是“用户调研结论”，而是 AI 按照通用软件方案自动补出来的**产品设计决策**或**技术设计决策**。其中有些可以作为后续设计候选项，但不能写成“来源于用户需求调研”。换句话说，这些需求**未被问卷证明存在**。如果直接塞进 SRS，会造成“需求来源不清”和“把设计当需求”的问题。 |
| AI 表述有歧义的需求 | 1. “基于教育邮箱或校园网的身份认证” 2. “线上支付/线下当面交易” 3. “订单状态追踪” 4. “送货至宿舍区” 5. “同校学生交易” 6. “推荐” | 人工调研更准确的说法应该是：用户需要的是**校内实名/学生认证**，并愿意接受**校内统一身份认证**，但问卷并没有限定一定是 **.edu 邮箱** 还是 **校园网认证**；交易方式上，用户更关心的是**同校区面交、跨校区代送/中转、校内配送、线上沟通自行约定**，而不是“线上支付”这一支付形态；用户痛点中突出的是**跨校区取货麻烦**，因此“送货至宿舍区”只是可能方案之一，不等于完整需求。 |                                                              |
