## 1. 系统概述

### 1.1 项目背景

校园二手交易平台是一个面向校内学生的二手商品交易系统,旨在为学生提供一个安全、便捷的二手商品发布和交易平台。通过该平台,学生可以发布闲置物品、浏览搜索商品、在线沟通、完成交易,并进行评价。

### 1.2 系统目标

1. **核心功能**: 提供完整的二手商品交易流程
2. **用户体验**: 简洁易用的界面,流畅的操作体验
3. **安全可靠**: 保证交易安全,用户信息安全
4. **可维护性**: 代码结构清晰,便于后期维护和扩展
5. **可演示性**: 10周内完成可运行原型

### 1.3 项目约束

- **开发周期**: 10周
- **团队规模**: 4人
- **技术背景**: 团队成员对Java和JavaScript有基础
- **项目性质**: 软件工程二课程项目

### 1.4 核心功能

| 功能模块 | 主要功能 |
|---------|---------|
| 用户管理 | 注册、登录、个人信息管理、头像上传 |
| 商品管理 | 发布商品、编辑商品、删除商品、商品图片管理 |
| 商品浏览 | 商品列表、商品搜索、商品详情、分类浏览 |
| 订单管理 | 创建订单、订单列表、订单详情、订单状态管理 |
| 评价系统 | 商品评价、评价列表、评分统计 |
| 消息系统 | 商品评论、评论回复 |
| 收藏功能 | 收藏商品、收藏列表 |
| 举报管理 | 举报商品/用户、举报处理 |

## 2. 架构设计

### 2.1 架构风格

本项目采用**前后端分离 + 分层架构 + MVC模式的模块化单体架构**。

详细决策理由参见 [ADR-001: 架构风格选择](./adr/ADR-001-架构风格选择.md)。

### 2.3 架构特点

1. **前后端分离**: 前端和后端独立开发、独立部署,通过RESTful API通信
2. **分层清晰**: Controller、Service、Repository职责明确,降低耦合
3. **模块化**: 后端按业务模块划分,边界清晰
4. **单体架构**: 所有模块部署在一个应用中,降低运维复杂度
5. **可演进**: 模块化设计为未来拆分服务保留空间

### 3.2 模块职责

**职责**: 用户认证、授权、个人信息管理

**核心类**:
- `UserController`: 用户接口控制器

- `UserMapper`: 用户数据访问

- `GoodImageService`: 商品图片业务逻辑

```
用户模块 ←───────┐

    ↓            │
```

## 6. 接口设计

### 6.1 API设计原则

1. **RESTful风格**: 使用HTTP动词表示操作(GET查询, POST创建, PUT更新, DELETE删除)
2. **统一响应格式**: 所有接口返回统一的JSON格式
3. **版本控制**: 通过URL路径进行版本控制(如/api/v1/)
4. **错误处理**: 统一的错误码和错误信息

### 6.2 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {
  }
}
```

### 6.3 核心API列表

#### 用户模块API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/users/register | 用户注册 |

本项目采用传统的三层部署架构:

```
┌─────────────────────────────────────────────────────────┐
│                      客户端层                             │
│  ┌─────────────────────────────────────────────────┐   │

└─────────────────────────────────────────────────────────┘
                          │
                          │ HTTPS
                          ↓
┌─────────────────────────────────────────────────────────┐
│                      应用层                              │
│  ┌─────────────────────────────────────────────────┐   │
│  │  Web服务器 (Nginx/Apache)                        │   │
│  │  - 静态资源服务                                   │   │
│  │  - 反向代理                                       │   │
│  │  - 负载均衡                                       │   │
│  └─────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────┐   │
│  │  应用服务器 (Tomcat embedded in Spring Boot)     │   │
│  │  - 运行Spring Boot应用                            │   │
│  │  - 处理业务请求                                   │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
                          │
                          │ JDBC
                          ↓
┌─────────────────────────────────────────────────────────┐
│                      数据层                              │
│  ┌─────────────────────────────────────────────────┐   │
│  │  MySQL数据库                                      │   │
│  │  - 数据持久化                                     │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

### 7.2 部署环境

#### 开发环境

- **前端**: Vite开发服务器 (localhost:5173)

- **数据库**: MySQL本地实例 (localhost:3306)

#### 生产环境

- **前端**: Nginx静态资源服务器
- **后端**: 独立服务器运行Spring Boot JAR包
- **数据库**: MySQL独立服务器

## 8. 安全架构

### 8.1 安全威胁分析

| 威胁类型 | 描述 | 风险等级 |
|---------|------|---------|
| SQL注入 | 通过输入恶意SQL语句攻击数据库 | 高 |
| XSS攻击 | 通过注入恶意脚本攻击用户浏览器 | 中 |
| CSRF攻击 | 伪造用户请求攻击 | 中 |
| 密码泄露 | 用户密码被窃取 | 高 |
| 数据泄露 | 敏感数据被非法访问 | 高 |

### 8.2 安全措施

#### 8.2.1 认证与授权

- **用户认证**: 用户名+密码登录
- **密码加密**: 使用BCrypt加密存储密码
- **会话管理**: 使用JWT Token管理用户会话
- **权限控制**: 基于角色的访问控制(RBAC)

#### 8.2.2 数据安全

- **SQL注入防护**: 使用MyBatis Plus参数化查询
- **XSS防护**: 前端输入验证和转义
- **CSRF防护**: 使用CSRF Token
- **HTTPS**: 生产环境使用HTTPS加密传输
- **数据加密**: 敏感数据加密存储

#### 8.2.3 接口安全

- **接口鉴权**: 所有接口需要验证Token
- **参数验证**: 后端参数校验
- **频率限制**: 接口访问频率限制
- **日志记录**: 记录关键操作日志

## 9. 性能考虑

### 9.1 性能目标

| 指标 | 目标值 |
|------|--------|
| 页面加载时间 | < 2秒 |
| API响应时间 | < 500ms |
| 并发用户数 | 100 |
| 数据库查询时间 | < 100ms |

### 9.2 性能优化策略

#### 9.2.1 前端优化

1. **代码分割**: 使用Vite进行代码分割,按需加载
2. **资源压缩**: 压缩JS、CSS、图片等资源
3. **懒加载**: 图片懒加载,路由懒加载
4. **缓存策略**: 合理使用浏览器缓存

#### 9.2.2 后端优化

1. **数据库优化**:
   - 为常用查询字段建立索引
   - 使用MyBatis Plus分页插件
   - 避免N+1查询问题
   
2. **代码优化**:
   - 使用连接池管理数据库连接
   - 合理使用缓存
   - 异步处理耗时操作

#### 9.2.3 数据库优化

1. **索引优化**: 为常用查询字段建立索引
2. **查询优化**: 优化慢查询SQL
3. **连接池**: 使用HikariCP连接池
4. **读写分离**: 未来可考虑读写分离
