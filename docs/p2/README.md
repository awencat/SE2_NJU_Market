#### ADR-001: 架构风格选择

**文件**: [ADR-001-架构风格选择.md](./adr/ADR-001-架构风格选择.md)

**决策**: 采用前后端分离 + 分层架构 + MVC模式的模块化单体架构

**核心理由**:
- 符合项目规模(中小型系统)
- 10周内可落地性最高
- 团队学习成本较低
- 便于模块划分和文档编写
- 保留后续演进空间

#### ADR-002: 技术栈选择

**文件**: [ADR-002-技术栈选择.md](./adr/ADR-002-技术栈选择.md)

**决策**: 
- 后端: Spring Boot 3.3.5 + Java 17 + MyBatis Plus 3.5.7
- 前端: Vue 3.5.32 + Vite 8.0.4
- 数据库: MySQL 8.0.32

**核心理由**:
- 快速开发和迭代
- 良好的开发体验
- 充足的社区支持和文档
- 团队学习成本可控

#### ADR-003: 数据库设计决策

**文件**: [ADR-003-数据库设计决策.md](./adr/ADR-003-数据库设计决策.md)

**决策**: 采用MySQL关系型数据库

**核心理由**:
- 数据关系复杂,关系型数据库天然适合
- 完善的事务支持
- 团队熟悉MySQL
- 成熟稳定,生态完善

## 架构概览

### 架构风格

**前后端分离 + 分层架构 + MVC模式的模块化单体架构**

```
前端(Vue 3) 
    ↓ HTTP/REST API
Controller层 
    ↓
Service层 
    ↓
Repository/Mapper层 
    ↓
数据库(MySQL)
```

### 核心模块

1. **用户模块**: 用户认证、授权、个人信息管理
2. **商品模块**: 商品发布、管理、浏览、搜索
3. **订单模块**: 订单创建、管理、查询
4. **评价模块**: 商品评价、评分
5. **评论模块**: 商品评论、回复
6. **收藏模块**: 商品收藏
7. **举报模块**: 举报管理
8. **管理模块**: 系统管理、举报处理

### 技术栈

**后端**:
- Spring Boot 3.3.5
- Java 17
- MyBatis Plus 3.5.7
- MySQL 8.0.32

**前端**:
- Vue 3.5.32
- Vite 8.0.4

## 架构原则

本项目架构设计遵循以下原则:

1. **适合项目规模**: 优先选择适合课程项目规模的架构,避免过度设计
2. **10周内可落地**: 所有决策必须考虑10周开发周期的约束
3. **团队学习成本**: 考虑团队成员的技术背景和学习曲线
4. **可维护性**: 优先保证代码清晰、结构合理
5. **可演进性**: 为未来可能的扩展保留空间

## 决策流程

1. **识别问题**: 发现需要做出架构决策的问题
2. **收集信息**: 调研技术方案,评估可行性
3. **团队讨论**: 通过架构辩论赛等形式讨论方案优劣
4. **记录决策**: 编写ADR文档,记录决策过程
5. **实施决策**: 在项目中实施架构决策
6. **回顾评估**: 定期回顾决策效果,必要时更新

## 演进路线

### 短期(0-6个月)

- 完善核心功能
- 优化用户体验
- 性能优化

### 中期(6-12个月)

- 引入Redis缓存
- 数据库读写分离
- 增加推荐算法

### 长期(12个月+)

- 模块拆分(如需要)
- 微服务化改造(如需要)
- 分布式部署
