## ADR模板

新的ADR应遵循以下模板:

```markdown
# ADR-XXX: [决策标题]

## 状态

[提议/已采纳/已废弃/已替代]

## 背景

描述导致此决策的背景和问题

## 决策

描述所做的决策及其理由

## 后果

### 积极影响

### 消极影响

### 风险

## 替代方案

列出考虑过的其他方案及其被拒绝的原因

## 参考

相关的文档、链接或讨论记录
```

## 决策原则

本项目架构决策遵循以下原则:

1. **适合项目规模**: 优先选择适合课程项目规模的架构,避免过度设计
2. **10周内可落地**: 所有决策必须考虑10周开发周期的约束
3. **团队学习成本**: 考虑团队成员的技术背景和学习曲线
4. **可维护性**: 优先保证代码清晰、结构合理
5. **可演进性**: 为未来可能的扩展保留空间

## 决策流程

1. **识别问题**: 发现需要做出架构决策的问题
2. **收集信息**: 调研技术方案,评估可行性
3. **团队讨论**: 通过架构辩论赛等形式讨论方案优劣
4. **记录决策**: 编写ADR文档,记录决策过程
5. **实施决策**: 在项目中实施架构决策
6. **回顾评估**: 定期回顾决策效果,必要时更新
